# 02. `get_unit_raw` 의 `expect` 메시지가 상황과 맞지 않음

## 증상

`src/main.rs:33~53` 의 `get_unit_raw` 함수에서 세 군데의 `.expect(...)` 메시지가 실제 실패 상황과 어긋나 있습니다.
컴파일은 되지만, 패닉이 났을 때 사용자에게 잘못된 안내가 나갑니다.

```rust
let value: f64 = parts
    .next()
    .expect("숫자를 입력해야 합니다.")    // (A) 첫 번째 토큰이 없을 때
    .parse()
    .expect("값을 입력해야 합니다.");      // (B) 첫 번째 토큰을 f64로 파싱 실패할 때

let unit_raw = parts.next().expect("숫자를 입력해야 합니다."); // (C) 두 번째 토큰(단위)이 없을 때
```

각 `expect`가 실제로 트리거되는 조건을 정리하면 이렇습니다.

| 위치 | 실패 조건 | 현재 메시지 | 문제 |
|------|-----------|--------------|------|
| (A) | 입력이 완전히 비어 있음 | "숫자를 입력해야 합니다." | OK |
| (B) | 첫 토큰이 숫자로 파싱 안 됨 (예: `abc kg`) | "값을 입력해야 합니다." | "값"이 모호. 실제로는 *숫자 형식* 문제 |
| (C) | 두 번째 토큰(단위)이 없음 (예: `10`만 입력) | "숫자를 입력해야 합니다." | 단위 누락인데 "숫자"라고 안내 |

특히 (C)가 명백한 오류입니다. 사용자는 이미 숫자를 입력했는데 "숫자를 입력해야 합니다"라는 메시지가 뜨면 혼란스럽습니다.

## 수정 방법

각 메시지를 실제 실패 조건에 맞게 바꿉니다.

**Before**
```rust
let value: f64 = parts
    .next()
    .expect("숫자를 입력해야 합니다.")
    .parse()
    .expect("값을 입력해야 합니다.");

let unit_raw = parts.next().expect("숫자를 입력해야 합니다.");
```

**After (제안)**
```rust
let value: f64 = parts
    .next()
    .expect("숫자를 입력해야 합니다.")
    .parse()
    .expect("숫자 형식이 올바르지 않습니다.");

let unit_raw = parts.next().expect("단위를 입력해야 합니다.");
```

문구 자체는 취향에 맞게 바꿔도 좋습니다. 핵심은 **각 `expect`가 트리거되는 실제 실패 조건과 메시지가 일치**하도록 만드는 것입니다.

## 참고 — 학습 관점

지금은 `expect`로 패닉을 내고 있지만, 사용자 입력 검증은 보통 `Result`를 돌려주는 형태로 바꾸는 게 일반적입니다.
다음 단계에서 `?` 연산자나 `Result<T, E>` 반환을 다루게 되면 자연스럽게 리팩터링 대상이 됩니다.
지금 단계에서는 메시지만 바로잡는 것으로 충분합니다.
