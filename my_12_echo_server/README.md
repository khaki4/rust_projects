### 12. 멀티스레드 TCP 에코 서버

Rust Book의 동시성(스레드) 챕터를 학습한 직후 1~2시간 안에 만들어 보는 네트워크 프로젝트입니다. 표준 라이브러리만 사용하며, 외부 크레이트를 추가하지 않는 것을 목표로 합니다.

#### 선행 챕터
rustlings 기준으로 다음 챕터를 마친 직후가 적합합니다.

- `20_threads` — `thread::spawn`으로 스레드 생성, `Arc<Mutex<T>>`로 여러 스레드 간 데이터 공유, mpsc 채널로 메시지 전달

#### 기능 명세
`127.0.0.1:7878`에서 TCP 연결을 listen합니다. 클라이언트가 접속할 때마다 새 스레드를 spawn해 받은 줄을 그대로 돌려보냅니다(에코). 활성 클라이언트 수를 `Arc<Mutex<usize>>`로 모든 스레드가 공유하며, 연결이 시작/종료될 때마다 값을 갱신합니다.

```bash
# 서버 시작 (인자 없음)
cargo run

# 별도 터미널에서 클라이언트 접속
nc 127.0.0.1 7878
```

서버 동작 요약

- `127.0.0.1:7878`에서 listen
- 연결 수락 시마다 새 스레드 spawn
- 받은 줄을 그대로 같은 연결로 echo
- 활성 클라이언트 수를 `Arc<Mutex<usize>>`로 공유, 연결 시 +1 / 종료 시 -1
- (옵션) mpsc 채널로 워커 스레드 → 메인 스레드 로그 메시지 전달

#### 핵심 활용 포인트
이 프로젝트에서 의식적으로 연습할 Rust 요소들입니다.

- `std::net::TcpListener` / `TcpStream` — `bind`, `accept`, `incoming()`으로 연결 수락
- `BufReader` / `BufRead::read_line` — 한 줄 단위로 입력 받기
- `thread::spawn` — 연결마다 워커 스레드 생성, `move` 클로저로 소유권 이전
- `Arc<Mutex<T>>` — 활성 클라이언트 수를 여러 스레드에서 안전하게 공유
- mpsc 채널(옵션) — 워커가 종료될 때 메인 스레드로 로그 메시지 전송

#### 검증 예시
다음 두 케이스가 기대대로 동작하면 1차 목표 달성으로 간주합니다.

```bash
# 1) 한 줄 보내고 그대로 돌아오는지 확인
#    터미널 A
cargo run
#    터미널 B
nc 127.0.0.1 7878
hello       # 입력
hello       # ← 같은 줄이 돌아오면 OK
```

```bash
# 2) 두 개의 nc 터미널을 동시에 띄워 각각 정상 동작하는지 +
#    서버 측 로그에서 활성 클라이언트 수가 1 → 2 → 1 → 0으로 갱신되는지 확인
#    터미널 B, C에서 각각 nc 127.0.0.1 7878
```

```bash
# 빌드만
cargo build

# 한 번에 빌드 + 실행 (서버 시작, 인자 없음)
cargo run

# 릴리스 빌드
cargo build --release
./target/release/my_12_echo_server

# 별도 터미널에서 클라이언트 접속 예시
nc 127.0.0.1 7878
```
